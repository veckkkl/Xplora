# Xplora

**Xplora** — нативное iOS-приложение для визуализации путешествий, ведения путевого дневника, просмотра хронологии поездок, планирования будущих направлений и отслеживания личной статистики исследования мира.

Проект разрабатывается на **Swift** с использованием **UIKit** и ориентирован на устройства iPhone.

---

## О проекте

Xplora помогает пользователю фиксировать и систематизировать личный опыт путешествий в одном приложении.

Приложение объединяет несколько пользовательских сценариев:

* просмотр интерактивной карты мира;
* ведение заметок о поездках с фотографиями, датами и локацией;
* создание хронологии совершённых поездок;
* ведение списка желаемых и посещённых стран;
* просмотр статистики путешествий;
* настройку профиля, языка, темы оформления и параметров приложения.

Цель проекта — дать пользователю простой инструмент, в котором путешествия можно не только хранить в виде отдельных записей, но и видеть как личную историю исследования мира.

## Основные возможности

### Карта

Экран карты является центральным экраном приложения.

Возможности:

* отображение интерактивной карты мира средствами MapKit;
* отображение аннотаций в местах, к которым привязаны заметки;
* просмотр превью заметки при нажатии на аннотацию;
* переход к полной заметке из превью;
* переход к списку заметок;
* создание новой заметки с экрана карты.

### Хронология

Экран хронологии отображает совершённые пользователем поездки.

Возможности:

* группировка поездок по годам;
* отображение страны, флага и диапазона дат поездки;
* отображение количества заметок, связанных с поездкой;
* ручное добавление новой поездки;
* редактирование дат поездки;
* удаление поездки;
* переход к заметкам конкретной поездки.

### Вишлист

Экран вишлиста предназначен для планирования будущих поездок и отслеживания уже посещённых стран.

Возможности:

* добавление страны из полного списка стран мира;
* добавление конкретного места внутри страны;
* разделение элементов на статусы «хочу посетить» и «посещено»;
* переключение статуса страны;
* удаление элемента из списка;
* отображение стран, которые есть в хронологии поездок, но ещё не добавлены в вишлист.

### Статистика

Экран статистики показывает прогресс пользователя на основе данных о поездках.

Возможности:

* отображение процента исследованного мира;
* отображение количества посещённых стран;
* отображение количества посещённых континентов;
* отображение статистики по отдельным континентам;
* автоматический пересчёт показателей при добавлении или удалении поездок.

### Заметки

Заметки используются как путевой дневник пользователя.

Возможности:

* создание заметки;
* редактирование заметки;
* удаление заметки;
* добавление заголовка и текста;
* добавление одной или нескольких фотографий;
* добавление локации;
* указание даты или диапазона дат;
* просмотр списка всех заметок;
* просмотр заметок, связанных с конкретной поездкой.

### Настройки и профиль

Раздел настроек содержит параметры приложения и данные пользователя.

Возможности:

* просмотр карточки пользователя;
* переход к экрану профиля;
* редактирование имени и аватара;
* выбор страны проживания;
* отображение динамического статуса пользователя;
* переключение видимости статуса;
* переключение темы оформления;
* выбор языка интерфейса;
* выбор типа карты;
* просмотр информации о приложении;
* просмотр политики конфиденциальности;
* удаление пользовательских данных.

## Архитектура

Проект построен с использованием подхода **Clean Architecture** и паттерна **MVVM**.

Также используется навигационный слой на основе паттернов:

* Coordinator;
* Builder;
* Router.

Основная идея архитектуры — разделить приложение на независимые слои:

```text
Presentation
↓
Domain
↓
Data
↓
Core
```

### Presentation

Слой пользовательского интерфейса.

Содержит:

* ViewController;
* ViewModel;
* UI-компоненты;
* экраны приложения;
* навигационную логику.

### Domain

Слой бизнес-логики.

Содержит:

* сущности предметной области;
* use cases;
* repository-протоколы.

### Data

Слой работы с данными.

Содержит:

* реализации репозиториев;
* локальные источники данных;
* сервисы для работы с платформенными возможностями.

### Core

Инфраструктурный слой.

Содержит:

* dependency injection;
* общие сервисы;
* вспомогательные компоненты.

## Локальное хранение данных

Xplora работает без собственной серверной части.

Пользовательские данные сохраняются локально на устройстве:

* заметки;
* фотографии, прикреплённые к заметкам;
* локации;
* поездки;
* элементы вишлиста;
* пользовательские настройки;
* данные профиля.

Для отображения картографических данных используется MapKit, поэтому для подгрузки тайлов карты может требоваться подключение к Интернету.

---

## Основные экраны

В приложении используется нижняя панель навигации с пятью вкладками:

1. **Вишлист** — список желаемых и посещённых направлений.
2. **Хронология** — список совершённых поездок.
3. **Карта** — интерактивная карта мира.
4. **Статистика** — прогресс исследования мира.
5. **Настройки** — профиль и параметры приложения.

Центральной вкладкой по умолчанию является экран карты.
